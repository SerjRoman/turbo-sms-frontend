1. save-dev - dev-dependecies - Модули которые нужны только для разработки (не пойдут в прод билд/бандл)
    Установка: npm i --save-dev PACKAGE_NAME
               npm i -D PACKAGE_NAME

2. При изменение пути к index.ts файлу(entry point), нужно поменять это в package.json.
    "main": "./src/app/index.ts"

3. Качаем библиотеку @expo/ngrok для работы в команде.

4. Создаём структуру. Для пустой папки можно вкинуть файлик .gitkeep для слежки. (just advice)

5. Path Alias - это опеределение путей в typeScript(проще говоря - позволяет определить более краткие маршруты к папкам/файлам)
    Для настройки идём в tsconfig.json:
        Напишем пример для папки shared.
            "paths": {
                "@shared/*": ["./src/shared/*"]
            }
            
            @shared/* - название алиаса(название для вашего пути).
            ["./src/shared/*"] - путь 
            @ - рандомный, уникальный знак. Можно любой(@/#/&).
            * - для указания, что можно вытянуть ВСЁ из этой папки(грубо говоря, вместо * любое название)
            

6. Стили стоит выносить в отдельный файл, а не оставлять в компоненте.

7. Нейминг файлов разделяется kebab-case (через -)
    home-page.tsx

8. Файлы со стилями мы называем *.styles.ts
    home.styles.ts

9. Файлы со стипизацией - это *.types.ts
    home.types.ts

10. Мы НЕ добавляем Barrel файл(index.ts) в Сегмент(ui, types, tools, api, constants).

11. Файл компонента называем с маленькой буквы kebab-caseом.

12. Если type при импорте пишем до {} - то всё, что в {} не будет в bundle.
    Если type при импорте пишем перед названием interface/type - то только этот interface/type помечается как.

Пропсы для TextInput.

autoComplete="email" - Подсказывает системе, что поле для email.
autoCorrect={false} - Отключает Т9.
inputMode="email" - Открывает специальную клавиатуру для email(@)
onChangeText={(text) => { - Функция, которая срабатывает при каждом вводе символа.
    console.log(text);
}}


flex - определяет пространство для элементов по ГЛАВНОЙ оси(в RN по умолчанию стоит flexDirection: "column")

Качаем библиотеку react hook form 
    npx expo install react hook form

При работе с react hook form есть пару моментов, которые нужно учитывать:
    1. Для управлениями полями(точнее для того чтобы связать компоненты с библиотекой) вместо register мы используем компонент Controller.
    2. При использовании useForm мы обязаны передать ему defaultValues(пример: input - пустая строка).
    3. Для textInput мы используем onChangeText вместо onChange.
    4. HandleSubmit вешается на форму, а не на кнопку. 

Controller Компонент
Нужен для связи какого-то компонента с библиотекой.
У него есть props:
    1. control - это prop, необхомый компоненту чтобы понять с каким хуком useForm он работает(то есть, с какой формой нужно связать компонент.)
    2. name - это имя для поля, которое нужно для компонента чтобы понять каким полем он управляет. (В typescript - это берётся из типизации формы.)
    3. render - этот prop настраивает взаимодействие между вашим полем и контроллером.
