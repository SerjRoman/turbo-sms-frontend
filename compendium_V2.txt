1. save-dev - dev-dependecies - Модули которые нужны только для разработки (не пойдут в прод билд/бандл)
    Установка: npm i --save-dev PACKAGE_NAME
               npm i -D PACKAGE_NAME

2. При изменение пути к index.ts файлу(entry point), нужно поменять это в package.json.
    "main": "./src/app/index.ts"

3. Качаем библиотеку @expo/ngrok для работы в команде.

4. Создаём структуру. Для пустой папки можно вкинуть файлик .gitkeep для слежки. (just advice)

5. Path Alias - это опеределение путей в typeScript(проще говоря - позволяет определить более краткие маршруты к папкам/файлам)
    Для настройки идём в tsconfig.json:
        Напишем пример для папки shared.
            "paths": {
                "@shared/*": ["./src/shared/*"]
            }
            
            @shared/* - название алиаса(название для вашего пути).
            ["./src/shared/*"] - путь 
            @ - рандомный, уникальный знак. Можно любой(@/#/&).
            * - для указания, что можно вытянуть ВСЁ из этой папки(грубо говоря, вместо * любое название)
            

6. Стили стоит выносить в отдельный файл, а не оставлять в компоненте.

7. Нейминг файлов разделяется kebab-case (через -)
    home-page.tsx

8. Файлы со стилями мы называем *.styles.ts
    home.styles.ts

9. Файлы со стипизацией - это *.types.ts
    home.types.ts

10. Мы НЕ добавляем Barrel файл(index.ts) в Сегмент(ui, types, tools, api, constants).

11. Файл компонента называем с маленькой буквы kebab-caseом.

12. Если type при импорте пишем до {} - то всё, что в {} не будет в bundle.
    Если type при импорте пишем перед названием interface/type - то только этот interface/type помечается как.
